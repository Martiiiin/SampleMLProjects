Number generator¶

A simple AI able to generate handwritten-like numbers Author: Martin Prieto Aguilera

We are going to make use of the MNIST dataset of handwritten numbers. Here, we build an autoencoder that will construct a reduced representation of the dataset.

Then, we shall obtain the statistics for the endodings of each number (the mean vector and the covatiance matrix for each number encoding) and use such statistics to define a gaussian generator that will provide us with the encodings of other images that, if done properly, should resemble the encodings of "handwritten" number associated to those statistics.

Finally, we shall pass the codings generated by the gaussian process through a decoder (which we shall train alongside the encoder) in order to obtain the images of new "handwritten" number (of course, in reality, they would be computer generated, but they will imitate the way humans write them).

Dataset preprocessing¶

Since we got a small dataset, we can do a general preprocessing and store the modified dataset on memory for latter use. First, we shall create a noisy dataset. We'll perform a min-max scaling and flatten input images.

Adding random noiseto the dataset is a way to build a more robust autoencoder, since it will make more difficult for it to simply memorize which images correspond to each number).

Although the best way to prevent a machine learning model from overfitting is simply to make it simple (this is, with a low capacity given the provided dataset). The simplier the model, the less patterns it can "memorize". THis way, we force it to learn the patterns that are truly useful for rebuilding the images later on.

Model design¶

Here, we design the Model using Keras interface

We shall define the autoencoder with two parameters:

  • The decrese rate which is the relationship between the number of neurons in layer k and layer k+1. A decrese rate of 2 means that each subsecuent layer has half the neurons that is predecessor.

  • The number of layers for encodig which is simply the depth of the encoder.

  • The decoder will be build symmetrically with respect to the encoder.

A part from the call() method (which will obtain the output of the whole model encoder-decoder) we shall implement a method specifically for encoding and a method specifically for decoding.

class MNIST5Autoencoder(tf.keras.Model):
    """
    A flat autoencoder. This one will start by flattening out the input images. The coder and decoder will have simmetry,
    but will not share any weights.
    
    Each dense leyer will reduce the dimmensionality of the previous layer by a factor of 2.
    """
    def __init__(self, input_units, decrease_rate=2, n_layers_enc=1):
        """
        decrease_rate: The number of hidden units in a coding layer divided by the number of hidden units in the previous layer
        n_layers_enc: The number of layers used for encoding (default 1)
        """
        super(MNIST5Autoencoder, self).__init__()
        
        # Define the number of units per hidden layer in codder
        divisor = decrease_rate**np.arange(1, n_layers_enc + 1, 1)
        units = (input_units / divisor).astype(np.int)
        
        # ENCODER
        self.codding_layers = []
        # Build codding layers
        self.codding_layers.append(tf.keras.layers.Dense(units[0], input_shape=(input_units,), activation=tf.keras.activations.elu))
        for u in units[1:-1]:
            layer = tf.keras.layers.Dense(u, activation=tf.keras.activations.elu)
            self.codding_layers.append(layer)
            
        # Append last codding layer (it shall use linear activation)
        self.codding_layers.append(tf.keras.layers.Dense(units[-1],activation=None))
            
        
        # DECODER
        self.decodding_layers = []
        # Build decodding layers
        for u in units[-2::-1]:
            layer = tf.keras.layers.Dense(u,activation=tf.keras.activations.elu)
            self.decodding_layers.append(layer)
        
        # Append last decoding layer (it'll be the same size as the input, with sigmoid activation)
        self.decodding_layers.append(tf.keras.layers.Dense(input_units, activation=tf.keras.activations.sigmoid))
        
        
    def call(self, X):
        """
        Forwardprop function.
        X: input data. Should be min-max normalized, to match a range of [0,1]
        """
        
        # ENCODING
        # Forwardprop through the first hidden layer
        Y = self.codding_layers[0](X)
        for layer in self.codding_layers[1:]:
            Y = layer(Y)

        # DECODING
        for layer in self.decodding_layers:
            Y = layer(Y)
        
        return Y


    def encode(self, X):
        """
        Encodex X
        """
        Y = self.codding_layers[0](X)
        for layer in self.codding_layers[1:]:
            Y = layer(Y)
      
        return Y

    def decode(self, X):
        """
        Decodes X
        """
        Y = self.decodding_layers[0](X)
        for layer in self.decodding_layers[1:]:
            Y = layer(Y)
      
        return Y

# Define an auxiliary function that shall aid us latter on
def random_choice_first_dimension(X, sample_size):
    """
    Selects several elements randomly in a 4-D tensor where the first dimension indexes the different elements.
    This is, if X is a vector of 3-D tensors, selects randomly a sample_size of such tensors.
    """
    selected_indices = np.random.randint(0, high=X.shape[0], size=sample_size)
    selector = np.zeros(X.shape[0], dtype=np.bool)
    selector[selected_indices] = True
    return X[selector,:,:,:]

These handwritten numbers have been written by no human!!¶

Here, we make the magic happens.

  • We obtain some statistics (the mean vector and the covariance matrix) for each of the numbers encodings.

  • The, we will build a gaussian generation process using the obtained statistics. Such process will give us encoded versions of the numbers from which we got the statistics that defined that process.

  • Finally, we will use the decode function of our model to build new images from the random generated encodings.

In [23]:
# We'll simulate some handwritten numbers
new_handwritten_numbers = []
# Obtain encodings for number all numbers
for i in range(10):
    encodings = model.encode(X_train_flatten[t_train == i, :])
    # Obtain mu and covariance
    mu_enc = np.mean(encodings, axis=0)
    mu_cov = np.cov(encodings, rowvar=False)
    # Generate random encodings sampled from a gaussian distribution with the computed parameters
    random_encodings = np.random.multivariate_normal(mu_enc, mu_cov, 5).astype(np.float32)
    # Decode the random_encodings to generate new handwritten numbers
    images_of_i = model.decode(random_encodings)
    images_of_i = np.reshape(images_of_i, (5,28,28,1))
    new_handwritten_numbers.append(images_of_i)
